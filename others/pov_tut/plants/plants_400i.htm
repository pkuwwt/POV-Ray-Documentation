 Alberi con POV-Tree e TOMTREE.inc      

POV-Tree é un 'tree generator' scritto in Java.
Autori: Tom Aust (tree algorithm) e Gena Obukhov (Java programming).
Ci sono le versioni localizzate disponibili: de, en, fr, it, ja, ru.
Formati di esportazione: Gli alberi si può esportare come POV-Ray include files con POV-Ray mesh o come include files, bisogno per TOMTREE-1.5.inc con i parametri per alberi di blob.
Il pacchetto include 12 diversi esempi di alberi.
Qualità: Gli alberi di POV-Tree sono buono per alberi di sfondo, con alcuni effords aggiuntivi così per più dettagliate vista Close-up, per seempio con le foglie HF e alberi con i fiori o con le frutta.
    Se si sta usando POV-Tree con uscita da esportazione come POV-Ray maglie, nota:
File con 'mesh' per gli alberi possono avere dimensioni da 10 a 20 MB o superiore - quindi abbiamo bisogno di essere paziente, se vogliamo caricare il file in un editor.
Dove trovare:
Il programma Java POV-Tree + TOMTREE-1.5.inc di Gena Obukhov.
Esempi molto interessanti si possono trovare sulla homepage di Tom Aust (il creatore dell'originale blob albero macro in 'tomtree.zip'):
http://www.aust-manufaktur.de/austv2x.html.
Non utilizzare il TOMTREE.inc originale perché la versione TOMTREE-1.5.inc è molto più veloce con le nuove versioni di POV-Ray.
POV-Tree funziona con la versione Java 1.4.2 o più recente.   (Per Java: Sun Microsystems.)
Installazione ed esecuzione:
Scarica il file (qui: povtree1.5.zip) e decomprimerlo in un direttorio a vostra scelta.
  Windows: Fare doppio clic sul file 'povtree.jar' per avviare il programma.
  Linux: riga di comando dal direttorio povtree decompressa: java -jar povtree.jar
POV-Tree finestra d'inizio:
Il modo migliore per imparare la gestione ottimale delle centinaia di dettagli di questo programma è quello di studiare il built-in esempi. Non dimenticate, si deve salvare tutti gli oggetti albero modificati con un nuovo nome, in modo da non perdere le impostazioni originali.
Aprire un albero di esempio:
Clicca sull'immagine in 2 ° fila, 2 ° colonna, 'Linden' e premere il tasto 'preview':

Salvare l'albero esempio come file include per POV-Ray "Linden.inc"
per un albero 'blob' con TOMTREE.inc:
Use: 'File' > 'Save as'

'File' > 'Save as'

Utilizzando l'albero del include file in POV-Ray:
Qui dobbiamo aggiungere qualcosa come le seguenti righe al nostro file di scena POV-Ray:
(Modificare i path name in base alle vostre path!) 
//-----------------------------------------------------
#include "povtree/trees/Linden.inc"
#include "povtree/TOMTREE-1.5.inc"
//-----------------------------------------------------
#declare Tree_01 = object{TREE double_illuminate hollow}
//-----------------------------------------------------
object{ Tree_01
        scale 4
        rotate< 0, 0, 0>
        translate< 0, 0.00, 0>
      } //--------------------------------------------
//----------------------------------------------------
Descrizione della scena per POV-Ray:
"Linden_01.pov" e "Linden.inc" ( TOMTREE-1.5.inc é parte di povtree1.5.zip )

L'immagine POV-Ray di 'Linden_01.pov'


"Linden.inc" - dichiara una serie di parametri per un albero blob con TOMTREE-1.5.inc,
Quando abbiamo includiamo questo set cosi:
#include "Linden.inc"
e poi cchiamano TOMTREE con
#include "TOMTREE-1.5.inc"
il questo file dichiara un oggetto denominato "Tree".
Questo oggetto ha una dimensione di circa 1 metro (ca. 3 ft),
così scalandolo da "4" scala produce un albero di dimensioni 4 metri.



    Le texture originali sono spesso un po 'scarso (può essere che sono buone con le precedenti versioni di POV-Ray), ma ci sono molti possibilties per modificarli a quelle più realistiche - vedi tra gli altri la mia pagina Textures per alberi di POV-Tree.
