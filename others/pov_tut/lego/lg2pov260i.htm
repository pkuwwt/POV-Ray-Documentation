Colori Adottati di Oggetti LGEO POV-Ray

Per una interpretazione corretta dei colori con POV-Ray il è necessario usare un contrasto standardizzato. L'adottamento corretto del contrast è detto "gamma correction".
La maniera di fare questo è differente in POV-Ray version 3.6 e nella nuova versione multicore 3.7
(attualmente solo come una versione beta, ma per in applicazioni standard funziona senza problemi seri).

In POV-Ray 3.6 dobbiamo aggiungere al inizio del testo di nostra scena 2 righe: 

#version 3.6
global_settings{assumed_gamma 1.0}
In POV-Ray 3.7 dobbiamo aggiungere al inizio del testo di nostra scena solo la riga: 
#version 3.7
 Per POV-Ray 3.6, 3.6.2, 3.7 insieme la riga: 
#if (version < 3.7) global_settings{ assumed_gamma 1.0 } #end
 (Niente di "gamma correction" nel file di scena più in version 3.7, queste chose sono (più consequente adesso) trasferito in file "POVRAY.INI" !)

I colori dei elementi LEGO virtuali per POV-Ray di LGEO sono definiti nel file incude "LGEO\lg\lg_colors.inc".
Alcuni colori si presenta fatto senza correzione gamma. Ciò è il est necessario di correggere i valori dei colori un poco. I colori qui sono anche definiti senza nessuno "normal" e senza "reflection" - in consequenza i mattoncini sembranno und poco smorto. Ma possiamo correggere questo con creare un nuovo file include per es. "lg_color2.inc". Dobbiamo memorizzare questo file nella stessa directory come il file originale. (Non dimenticare chambiare anche il nome nel file include e/o file di scena della vostra scena!).

Allora, come vuole i colori è vostra decisione absolutamente libera - Le piace una testura di tipo zebra o qualcos'altro pazzo? - Libertà senza confini! - ma penso la versione qui in alto più destra mi sembra di essere la versione più "realistico" come il rosso di plastica classico dei mattoncini LEGO®.
Non ho riveduto tutto il set dei colori di file include LGEO adesso. Ma potete download qui il mio primo esperimento di correzione del file include color lg_color2.inc per vedere i mie modificazioni.
