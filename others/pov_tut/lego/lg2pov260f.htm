Adapter les Couleurs des Objecs POV-Ray de LGEO

Pour correcter interprétation des couleurs de POV-Ray est nécessaire utiliser un contraste standardisé. L'adaptation correct du contraste on appele correction de gamma.
La manière de faire cela est différent en POV-Ray version 3.6 et avec la version 3.7 nouveau multicore.
( actuellement seulement une version beta, ma cela marche très bon !).

En POV-Ray 3.6 nous devons ajuter à la tête du texte de scène 2 lignes : 

#version 3.6
global_settings{assumed_gamma 1.0}
En POV-Ray 3.7 nous devons ajuter à la tête du texte de scène seulement 1 ligne: 
#version 3.7
 Pour POV-Ray 3.6, 3.6.2, 3.7 la ligne: 
#if (version < 3.7) global_settings{ assumed_gamma 1.0 } #end
 (Pas de correction de gamma plus dans le fichier de scène en version 3.7, cette choses sont (plus consequent maintenant) dans le fichier "POVRAY.INI" !)

Pour les éléments virtuels de LEGO pour POV-Ray de LGEO les couleurs sont défini dans le fichier incude "LGEO\lg\lg_colors.inc".
Quelques-unes de ces couleurs sont défini sans aucune corrections de gamma. Pour cette raison il est nécessaire de correcter les valeurs un peu. Ces couleurs standard sont aussi défini sans " normal " et sans " reflection " à cause de cela les briques semblent un peu trop mat. Nous pouvons corriger cela par créer un nouveau fichier de couleurs par ex. "lg_color2.inc". Nous devons sauvegarder ce fichier dans le même régistre de fichier original. (N' oubliez pas échanger il nome dans le fichier de scène et / ou dans le fichier include correspondant de votre scène!).

POV-Ray3.6 avec correction de gamma

Alors, comme vous voulez apparaître les couleurs est tout votre décision libre - si vous voulez on peut aussi utiliser une texture de zebra ou autre choses bizarre - les possibilités sont illimité! - mais je pense que la version ici en haut à droite semble plus " réaliste " comme le rouge plastique classique de briques LEGO.
Je n'ai pas revue et corrigée tout la série de couleurs du fichier include LGEO. Mais vous pouvez télécharger mon premier fichier de couleurs corrigé ici : lg_color2.inc pour voire les changements dans ce fichier.
